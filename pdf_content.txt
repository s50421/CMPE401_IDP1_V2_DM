2.  Background & Motivation 
Real-time object detection in computer vision plays a central role in: 
• Autonomous driving 
• Robotics 
• Smart surveillance 
• Industrial inspection 
• Smart cities 
The YOLO (You Only Look Once) family, maintained by Ultralytics, is one of the most widely used 
real-time detection frameworks in industry. 
Modern versions such as YOLOv11 (and recent updates such as YOLOv26) aim to improve: 
• Speed–accuracy trade-off 
• Small-object detection capability 
• Model efficiency 
• Training stability 
This project focuses on building a complete experimental pipeline using YOLOv11 (or YOLOv26), 
performing structured experiments, systematic analysis, and multi-version comparisons. 
https://docs.ultralytics.com/models/yolo11/ 
https://docs.ultralytics.com/models/yolo26/ 
https://github.com/ultralytics/ultralytics

3. Project Objectives 
Students will: 
1. Understand the YOLO detection pipeline 
2. Fine-tune a modern YOLO model 
3. Design and execute structured experiments 
4. Analyze training and validation loss behavior 
5. Diagnose overfitting and underfitting 
6. Perform iterative model improvements 
7. Compare multiple YOLO versions quantitatively 
8. Present reproducible research results via Github

6. Project Summary 
This project requires students to design, train, analyze, and systematically evaluate a modern object 
detection system using YOLOv11 (or YOLOv26). The focus is not only on model performance, but 
on understanding training dynamics, experimental design, and comparative analysis. 
Students must: 
• Build a complete detection pipeline 
• Analyze training and validation loss behavior 
• Diagnose overfitting and underfitting 
• Conduct controlled experiments 
• Perform theory-driven improvements 
• Compare multiple YOLO versions quantitatively 
• Present reproducible results via GitHub 
The goal is to develop both engineering implementation skills and a rigorous understanding of modern 
deep learning systems through structured experimentation and critical analysis.
